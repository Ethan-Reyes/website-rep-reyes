## Reporting a Vulnerability

I take the security of my projects seriously. If you discover a security vulnerability within this repository or the live site, please **do not open a public issue.** Publicly disclosing a vulnerability can put the project and its data at risk.

Instead, please report the issue privately:

1. **Email:** [Insert Your Professional Email Here]
2. **Subject:** SECURITY VULNERABILITY - [Project Name]

### What to include:
* A brief description of the vulnerability.
* Steps to reproduce the issue.
* Potential impact.

I am currently a student at **Wichita State University**, but I will acknowledge your report within 48-72 hours and provide a timeline for a fix.

## Policy on Data
This project utilizes a Netlify ETL pipeline for visitor analytics. No personally identifiable information (PII) is intentionally stored. If you believe PII is being exposed, please follow the reporting steps above.
